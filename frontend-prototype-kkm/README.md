# 프론트엔드 시안 (KKM)

서버 예약·할당 관리 시스템 **프론트엔드 부**의 화면 시안과 정보구조(IA).
백엔드 API와 서버 풀 데이터가 화면에 어떻게 노출되는지를 설계 관점에서 보여주는 것이 목적이며,
실 서비스 디테일보다 **소프트웨어 설계·AIOps 동작**을 드러내는 데 초점을 둡니다.

## 파일

| 파일 | 내용 |
|------|------|
| [`index.html`](./index.html) | 역할별 화면 시안 (정적 HTML, 브라우저로 바로 열람) |
| [`frontend-ia.drawio`](./frontend-ia.drawio) | 정보구조(IA) — 화면 → 컴포넌트 → API(F·UC) → 서버풀/스케줄러 흐름 |

## 디자인 방향

- **스타일 A** — 사이드바 + 고밀도 테이블 (운영툴 톤), 라이트 테마, Pretendard. 기존 설계 문서 사이트와 톤 일치.
- **AI 티 배제** — 전문적·심플. 과장된 그래픽 없이 상태/현황이 먼저 읽히도록.

## 구성

상단 **역할 스위처**로 세 역할을 전환합니다.

- **STU 학생·연구원** — 서버 현황 · 예약 신청(충돌 시 대안) · 내 예약 · 알림
- **MGR 팀 관리자** — 승인함(부분 승인) · Quota 관리 (+ STU 기능)
- **ADM 서버 관리자** — 운영 대시보드(AIOps 파이프라인) · 인시던트·AIOps · 서버 상태·가용성(90일 가동 이력) · 서버 관리 · 계정 보안

## 설계가 드러나는 장치

- **설계 추적 바** — 화면마다 매핑되는 API · 기능(F) · UC · 엔티티를 한 줄로 표기.
- **UC 뱃지** — `UC17 · 승인 타임아웃`처럼 UC 번호 + 기능명.
- **AIOps 파이프라인** — `수집(UC14) → 탐지(UC18) → 상관(UC24) → 예측(UC22·23) → 설명(UC25) → 자동복구(UC15·16)`. 구현/제안 단계를 구분 표기.
- **용어 툴팁** — health, MTBF/MTTR, μ±2σ, riskScore, Holt-Winters 등에 점선 밑줄 + hover 설명.

> 데이터는 전부 합성 목 데이터입니다(실 데이터 아님).
