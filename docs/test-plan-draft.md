# 테스트 계획 — 초안

> 작성 2026-05-29 · 검토용. NFR(`dynamic-models-nfr-draft.md` §3)을 검증하는 테스트 전략. 구현 단계에서 실제 케이스로 확장한다.
> 범위: 백엔드(FastAPI + APScheduler + PostgreSQL + Redis) 중심, 프론트는 E2E 일부.

## 1. 테스트 레벨

| 레벨 | 대상 | 도구(후보) |
|---|---|---|
| 단위 | core/services 도메인 로직(Quota 계산, health score 공식, μ±2σ 판정) | pytest |
| 통합 | API ↔ DB ↔ Redis (실 DB 기동) | pytest + testcontainers(PostgreSQL/Redis) |
| E2E | 주요 사용자 흐름(예약→사용→반납, 승인) | Playwright(프론트) |
| 정적 분석 | 코드 품질·타입·보안·의존 방향 | ruff, mypy, bandit, import-linter |
| 동적 분석 | 런타임 성능·메모리·쿼리 | py-spy/cProfile, tracemalloc, EXPLAIN, pytest-cov |

## 2. 기능 테스트

- UC별로 **주 시나리오 + 대안(A) + 예외(E)** 를 케이스화. 예: UC04 → 정상 예약 / Quota 초과 분기(A1) / 락 충돌(E1) / MAINTENANCE 전환(E2).
- **권한 매트릭스**: (STU·MGR·ADM) × (각 엔드포인트) 표로 허용/403 전수 검증 → NFR-S1·S3.

## 3. 동시성 테스트 (핵심 차별점)

| 케이스 | 시나리오 | 통과 기준 | NFR |
|---|---|---|---|
| 이중 예약 | N개 스레드가 동일 서버 동시 예약 | 정확히 1건 성공, 나머지 409 | NFR-C1 |
| 취소·회수 경합 | 직접 반납 vs 자동 회수(UC15) 동시 | 1건만 반영, 충돌 측 안전 종료 | NFR-C1·R1 |
| 승인·타임아웃 | 승인(UC09) vs 자동 거절(UC17) 동시 | 1건만 상태 전이 | NFR-C2 |
| 만료·반납 | 만료(UC16) vs 직접 반납(UC07) 동시 | 1건만 처리 | NFR-C1·R1 |

기법: `threading`/`asyncio.gather`로 동시 호출, 또는 DB 배리어로 race window 강제.

## 4. 성능·부하 테스트

> 도구: k6 또는 locust. 대상: 조회(NFR-P1)·예약(NFR-P2)·스케줄러(NFR-P3).

| 유형 | 목적 | 방법 | 합격선 |
|---|---|---|---|
| **부하(Load)** | 예상 정상 부하에서 SLA 충족 | 동시 50명·정상 RPS 유지 | p95 ≤ 목표(NFR-P1·P2) |
| **스트레스(고부하)** | 정상 초과 시 열화·회복 거동 | 정상의 2~5배까지 점증 | 에러율 급증 지점·복구 확인 |
| **중단점(Breakpoint/Capacity)** | 한계점(동시 사용자/RPS) 탐색 | 부하 점증하며 SLA 깨지는 지점 측정 | 한계 수치 문서화(NFR-Sc1) |
| **소크/내구성(지속성)** | 장시간 지속 부하에서 누수·저하 | 정상 부하 수 시간 지속 | 메모리·p95 안정, 버퍼 복구(NFR-R2) |

부가: 메트릭 수집은 서버 100대 모사(server-pool 컨테이너 N개)로 1주기 ≤ 30s 검증(NFR-P3).

## 5. 보안 테스트

- 인가 매트릭스(NFR-S1·S3), 수평 접근(타팀 코드 노출 검사).
- Rate limit/계정 잠금(NFR-S2): 1분 내 임계 초과 → 15분 잠금, 잠금 중 429, ADM 수동 해제.
- 감사 로그(NFR-S4): ADM/MGR 조회·삭제·잠금 시 로그 적재 검증.

## 6. 정적 분석 (코드 작성과 병행)

| 도구 | 검사 | 연계 |
|---|---|---|
| ruff | 린트·스타일 | NFR-M1 |
| mypy | 타입 안정성 | NFR-M1 |
| bandit | 보안 취약(SAST) | NFR-S* |
| import-linter | api→core→infra 단방향 의존 강제 | NFR-M1, ADR-04 |
| pytest-cov | 커버리지 임계(예: ≥ 70%) | 전반 |

## 7. 동적 분석

- 프로파일링: py-spy / cProfile로 핫스팟(예약·대시보드 집계) 측정.
- 메모리: tracemalloc으로 소크 테스트 중 누수 추적(NFR-R2).
- 쿼리: 핵심 경로 EXPLAIN(ANALYZE)로 인덱스·N+1 점검(메트릭 시계열·대시보드 집계).
